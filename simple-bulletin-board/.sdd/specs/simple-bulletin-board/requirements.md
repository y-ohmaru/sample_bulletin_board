# 要件定義書

## 機能概要
Spring Boot、H2 Database、Thymeleaf を使用した簡易掲示板アプリを実装する。利用者は `/posts` にアクセスして投稿一覧を確認し、名前と内容を入力して投稿を登録できる。また、一覧画面から既存投稿を削除できる。入力時には必須チェックと文字数制限を行い、エラー時は画面上にメッセージを表示する。

## ユーザーストーリー
- ユーザーとして、掲示板の投稿一覧を見たい。なぜなら現在の投稿内容をすぐに把握したいから。
- ユーザーとして、名前と内容を入力して投稿したい。なぜなら新しい情報や意見を掲示板に追加したいから。
- ユーザーとして、不要な投稿を削除したい。なぜなら一覧を最新で見やすい状態に保ちたいから。
- ユーザーとして、入力エラーを画面上で確認したい。なぜなら不正な入力をその場で修正したいから。

## 機能要件
### 要件1：投稿一覧表示
- `/posts` への GET リクエストで投稿一覧画面を表示する。
- 一覧画面は `posts/list.html` を使用する。
- 受入基準：
  - [ ] `GET /posts` にアクセスすると投稿一覧画面が表示されること
  - [ ] 保存済みの投稿が一覧に表示されること
  - [ ] 各投稿について識別可能な情報が画面上で確認できること

### 要件2：投稿登録
- ユーザーは名前と内容を入力して新しい投稿を登録できる。
- 登録処理は POST リクエストで受け付け、成功後は一覧画面に戻る。
- 受入基準：
  - [ ] 名前と内容を送信すると新しい投稿が保存されること
  - [ ] 登録成功後に一覧画面へ戻ること
  - [ ] 登録した投稿が一覧画面に反映されること

### 要件3：投稿削除
- ユーザーは一覧画面から既存投稿を削除できる。
- 削除処理は POST リクエストで実行する。
- 受入基準：
  - [ ] 一覧画面から対象投稿の削除操作を実行できること
  - [ ] 削除した投稿が保存データから除去されること
  - [ ] 削除後の一覧画面に対象投稿が表示されないこと

### 要件4：入力バリデーション
- 投稿時に名前は必須入力、内容は必須入力かつ100文字以内とする。
- バリデーションエラー時は画面上にメッセージを表示し、修正可能な状態を維持する。
- 受入基準：
  - [ ] 名前が未入力の場合は登録されず、エラーメッセージが表示されること
  - [ ] 内容が未入力の場合は登録されず、エラーメッセージが表示されること
  - [ ] 内容が100文字を超える場合は登録されず、エラーメッセージが表示されること
  - [ ] バリデーションエラー時に入力内容を見直せる画面が表示されること

### 要件5：データ永続化
- 投稿データは H2 Database に保存する。
- アプリ起動時に投稿テーブルを自動作成する。
- 受入基準：
  - [ ] 投稿データが H2 Database に保存されること
  - [ ] アプリ起動時に必要なテーブルが利用可能な状態になること

## 非機能要件
- パフォーマンス：一覧表示、登録、削除がローカル開発環境で支障なく実行できること
- セキュリティ：機密情報をソースや文書に含めないこと
